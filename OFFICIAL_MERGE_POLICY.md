# 官方 Skins Pro 合并策略

## 目标

在跟随官方新功能的同时，保护本 fork 的平板体验和自定义功能。

## 固定流程

`upstream/master` 已被官方压缩为独立历史，和本 fork 没有共同祖先。因此官方代码只能作为功能快照，禁止直接 merge、rebase、reset 或覆盖本 fork。

1. 获取官方最新快照：`git fetch upstream`。
2. 对比目标功能涉及的文件，不对整个仓库做合并。
3. 对照 `CUSTOM_FEATURES.md` 标记影响范围：HA 实体适配、摄像头、门禁、媒体、kiosk 和 God of War 主题均为保护范围。
4. 只手动移植独立、通用的功能；每个功能单独提交，例如搜索、设备控制、翻译或编辑器体验。
5. 有冲突的功能先出方案，不直接覆盖；涉及保护范围时默认不移植。
6. 每次移植后执行 `npm run type-check`、`npm run build` 和 `UPGRADE_CHECKLIST.md`。
7. 通过本地验证后部署到 HA；God of War、摄像头、门禁和当前使用的设备必须人工确认。
8. 用户确认 HA 效果后，才推送到 GitHub fork `zhouningha/znSkins-Pro`。推送前先处理 `origin/master` 的远端变更，不强推。

## 主题策略

- `skins-pro/god_of_war_3_wall` 和现有自定义主题是稳定主题，默认不改动。
- 官方主题只能作为可选资源导入当前 fork；其布局、摄像头和安防页面必须单独测试。
- 官方主题错位时，复制为 `*-zn` 适配主题并做主题专属修正；禁止以全局 CSS 修复影响 God of War 或其他主题。
- 不把官方主题目录或官方构建产物覆盖到本 fork 的 `skins-pro/`、`dist/`、`store/`。

## 决策优先级

1. HA 能正常访问
2. 平板 kiosk 稳定
3. 已登记定制功能不丢
4. God of War 主题视觉一致
5. 官方新增功能

## 禁止事项

- 禁止直接用官方包覆盖 HA 生产目录。
- 禁止同时加载官方版和 fork 版 Skins Pro。
- 禁止未备份就修改 HA 生产文件。
- 禁止为了合并官方功能删除已登记的定制功能。
- 禁止发现冲突后直接试错式乱改。
